RESTful TODO API на Go для демонстрация навыков backend-разработки.

Ключевые технологии и подходы:
- Создание, чтение и удаление задач через эндпоинты REST API.
- Чистая архитектура с разделением на handlers, models, database
- Интеграция с базой данных PostgreSQL с использованием GORM для ORM и автоматических миграций.
- Маршрутизация через легковесный роутер chi
- Базовая обработка ошибок и ответы в формате JSON.
- Конкурентобезопасная работа с базой данных
- Unit-тесты с проверкой краевых случаев
- Полная контейнеризация приложения и базы (Docker + Docker Compose)
- Автоматизация тестирования, линтинга и деплоя через GitHub Actions ( Пайплайн CI/CD через GitHub Actions для тестирования и пуша Docker-образов.)


Стек Технологий:
- Язык: Go 1.25
- База данных: PostgreSQL (с GORM для ORM)
- Роутер: github.com/go-chi/chi/v5
- Тестирование: Встроенный пакет тестирования Go
- Контейнеризация: Docker, Docker Compose
- CI/CD: GitHub Actions
- Другие зависимости: Смотрите `go.mod` для полного списка


Использование:
API предоставляет следующие эндпоинты:

- GET /tasks: Получить все задачи.
  - Ответ: Массив JSON с задачами (например, `[{"id":1,"description":"Задача 1","note":"Детали"}]`)

- POST /tasks: Создать новую задачу.
  - Тело запроса: Объект JSON (например, `{"description":"Новая задача","note":"Опциональная заметка"}`)
  - Ответ: 201 Created при успехе.

- GET /task/{id}: Получить задачу по ID.
  - Ответ: Объект JSON с задачей.

- DELETE /task/{id}: Удалить задачу по ID.
  - Ответ: 200 OK при успехе.



Тестирование:

Проект включает unit-тесты в `handlers/handlers_test.go` для проверки поведения обработчиков (например, некорректный JSON, пустое тело, некорректные ID и тд).
Некоторые тесты требуют подключенной базы данных; они будут пропущены, если БД недоступна.



CI/CD:

Настроен workflow GitHub Actions (`Golang-pip`):
- При пуше: Запускает тесты (`go test`) и проверку (`go vet`).
- При тегах: Собирает и пушит Docker-образ в Docker Hub (используя секреты `ID_DOCKERHUB` и `TOKEN_DOCKERHUB`).
